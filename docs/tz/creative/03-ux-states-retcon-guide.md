# ТЗ (Creative) — UX состоятния (loading/error/empty/…) и гайд по ретконам

Цель: проработать пользовательские тексты/состояния интерфейса для всех ключевых экранов Mini App и подготовить прикладной гайд по ретконам для редакторов/гейм‑мастеров.

## Экраны и состояния
- Общие состояния: `loading | ready | empty | error | unauthorized`.
- Разделы:
  - Auth/Config: тексты приветствия, проверка initData, ошибки обмена JWT.
  - Generation: профили/деталь профиля/создание результата — тексты для `loading/error/empty`, подписи к полям, хинты.
  - Knowledge Search: плейсхолдеры, `no results`, подсказки по тегам.
  - Media Jobs: постановка задачи, ожидание, успех (CDN URL), ошибка/повтор.
  - Party Sync (WS): подключение/потеря соединения/повторное подключение, сообщение‑плейсхолдер.

## Формат поставки
- Таблица/файл с текстами UI: `docs/ux/strings.json` (ключ → локализованная строка, `ru-RU`).
- Справочник состояний по разделам: `docs/ux/states.md` — что и где показывать; требования к длине и тону.
- Скрин‑наброски (не обязательно финдизайн): pdf/png.

## Гайд по ретконам (для редакторов)
- Типовые ситуации: неверная схема, противоречие лору, лишняя длина, нежелательный тон, неоднозначность.
- Набор заготовок для уточняющих подсказок (prompt‑hints) в 1–2 предложения, с ссылкой на поле/ограничение.
- Примеры до/после (минимум 5).

## Критерии приёмки
- Полный набор строк в `docs/ux/strings.json`; проверка покрытия — есть строки для каждого состояния и раздела.
- `states.md` описывает поведение для всех фич; разработчик может напрямую интегрировать тексты.
- Гайд по ретконам содержит примеры и правила, согласованные с профилями (см. Creative 02).

## Что должна вернуть команда
- `docs/ux/strings.json` — JSON со строками вида:
  - `app.header.status.loading`, `app.header.status.ready`, `auth.status.checking`, `auth.status.authenticated`, `auth.error.exchange_failed`,
  - `generation.loading`, `generation.empty`, `generation.error`, `media.loading`, `media.succeeded`, `media.error`,
  - `search.placeholder`, `search.no_results`, `ws.status.connected/disconnected/error` и пр.
- `docs/ux/states.md` — перечень состояний по каждому экрану с требованиями к тону/длине.
- (Опционально) `docs/ux/wireframes/*.png` — быстрые скетчи экранов.
